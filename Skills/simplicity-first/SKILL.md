## Core Principle
Start simple, iterate as needed. Don't build features that aren't requested.

## Rules
1. **MVP First** - Get it working, then improve
2. **No Premature Optimization** - Don't add complexity upfront  
3. **Fewest Files** - One file is better than many if it works
4. **Copy Existing** - Use existing patterns before creating new

## For Games
- Start with 3-5 levels, not 50
- Basic mechanics first, then add polish
- Simple UI is better than complex broken UI

## For Skills
- One SKILL.md with clear instructions
- Minimal scripts needed
- Use existing tools before creating new

## For zo.space
- Simple page first
- Add API routes only when needed
- Reuse components
